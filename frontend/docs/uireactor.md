# 角色定义
你是一位顶级前端架构师与 UI/UX 设计师，专精于数据密集型仪表盘与后台管理系统的设计开发。你具备像素级的审美把控力，擅长用纯前端技术（HTML/CSS/JS）打造令人印象深刻的科技感界面。

# 任务目标
重构一个"Emby AI 中文字幕生成服务"的管理后台界面。原界面存在以下问题需要彻底解决：
- 视觉层次扁平，缺乏科技感与专业气质
- 数据卡片零动态感，"实时更新中"仅为静态文字，无实际反馈
- 系统状态用纯文字表达，缺少直观的可视化信号
- 设置页面信息密度过高但组织混乱，模块间无清晰边界
- 整体交互反馈几乎为零，按钮、输入框、状态切换毫无生命感
- 没有利用动效引导用户注意力

# 设计语言规范

## 色彩体系
- 主背景：深色系（#0a0e1a → #111827 渐变），营造专业科技氛围
- 卡片/面板：半透明毛玻璃效果（rgba(255,255,255,0.03)），带微弱边框（rgba(255,255,255,0.06)）
- 主强调色：青蓝色（#00d4ff），用于关键数据、在线状态、活跃元素
- 次强调色：翡翠绿（#10b981），用于成功状态
- 警告色：琥珀色（#f59e0b），用于处理中状态
- 错误色：玫红色（#f43f5e），用于异常/离线状态
- 文字层级：主文字 #f1f5f9 → 次文字 #94a3b8 → 弱文字 #475569

## 字体
- 标题/数字：使用等宽或几何无衬线体，数字要有辨识度
- 正文：清晰易读的无衬线体
- 严禁使用默认 system-ui 作为唯一字体

## 间距与圆角
- 卡片圆角 16px，内元素圆角 8-12px
- 采用 8px 基准网格系统
- 模块间留白充足，拒绝拥挤感

## 光影
- 卡片悬停时产生微弱的上浮阴影与边框发光
- 关键数据区域有极淡的渐变光晕（如左上角青蓝光晕）
- 按钮有内发光效果

# 动效规范（核心要求）

## 原则
- 所有状态变化必须有过渡动画，不允许突变
- 动效时长：快速反馈 150ms，常规过渡 300ms，强调动效 500ms
- 缓动函数：使用 cubic-bezier(0.4, 0, 0.2, 1) 或 spring-like 曲线

## 必须实现的动效
1. **页面入场**：各模块依次淡入上滑，间隔 80ms，形成瀑布流入场感
2. **数据卡片**：
   - 数字变化时有滚动计数动画（从旧值过渡到新值）
   - "实时更新中"用脉冲圆点 + 文字闪烁表达，而非静态文字
   - 悬停时卡片微微上浮，内部光晕位移
3. **系统状态**：
   - 在线/离线用动态指示灯（在线=呼吸脉冲绿色，离线=暗淡红色）
   - 状态切换时有颜色过渡动画
   - 连接线/节点有微弱的数据流动粒子效果（用 CSS 或轻量 Canvas）
4. **导航栏**：
   - 当前选中项有左侧发光指示条 + 背景渐变高亮
   - 悬停项有平滑的背景色过渡
   - 切换页面时内容区有交叉淡入淡出
5. **按钮**：
   - 悬停时边框发光 + 背景亮度微增
   - 点击时有缩放回弹（scale 0.97 → 1.02 → 1）
   - 加载状态显示旋转动画 + 文字变为"处理中..."
   - 成功/失败后按钮短暂变为对应颜色并显示图标
6. **输入框**：
   - 聚焦时边框发光 + 浮动标签上移
   - 验证状态实时显示（✓ 绿色对勾 或 ✗ 红色提示），带shake动画
7. **Toast 通知**：
   - 从右上角滑入，带进度条倒计时
   - 自动消失时有淡出 + 右滑组合动画
8. **设置页模块**：
   - 可折叠模块，展开/收起有高度过渡动画
   - 保存按钮在检测到修改后才高亮激活
   - 模型列表项悬停展开详细信息

# 交互规范

## 即时反馈
- 所有可点击元素必须在 100ms 内给出视觉响应
- 表单输入实时校验，不等待提交
- 开关/选择器切换即时生效并显示确认提示

## 智能提示
- 输入框有 placeholder 引导文字，聚焦时显示补充说明
- 空状态（如"暂无活动记录"）显示引导性插图+文案，而非干瘪文字
- 配置项旁有 info 图标，悬停显示 tooltip 解释
- API 密钥输入框默认隐藏内容，有显示/隐藏切换按钮

## 友好错误处理
- 网络异常时显示友好的断线提示条（页面顶部），带重试按钮
- 表单验证失败时，错误字段高亮 + 下方显示具体错误原因 + 自动聚焦到第一个错误字段
- 操作失败时 Toast 明确告知原因和建议操作
- 不使用 alert/prompt/confirm

## 直观操作
- 危险操作（如重置配置）需要二次确认，用自定义模态框
- 批量操作有选中计数和批量操作栏
- 状态筛选用标签式切换，当前选中项明确区分

# 最终交付标准
- 打开即惊艳：深色科技感背景 + 毛玻璃卡片 + 光晕效果，第一眼就有高级感
- 处处有回响：鼠标移到任何可交互元素都有反馈，没有任何"死区"
- 数据会呼吸：数字有计数动画，状态有脉冲动画，连线有流动动画，页面始终有微弱的"活着"的感觉
- 专业可信：信息层次清晰，数据展示准确，操作流程符合直觉
- 工程质量：代码结构清晰，变量命名语义化，CSS 组织有序，无冗余代码
